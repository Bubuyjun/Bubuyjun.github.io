从零开始的免费Hexo博客搭建教程

前言

哪个男孩不想拥有一个属于自己的博客网站呢?
但是考虑到大部分人家 境 贫 寒的现状 特此出一篇免费搭建博客网站的教程
注意:文中提供的链接皆为国内源,官方链接我会补充在文末

什么是Hexo

Hexo 是一个快速、简洁且高效的博客框架。Hexo 使用 Markdown（或其他渲染引擎）解析文章，在几秒内，即可利用靓丽的主题生成静态网页。
你可以把生成的网页挂在很多地方,服务器,OSS(对象存储,但是我没用过),以及接下来要用的GitHub Pages等Pages服务都可以.

各个开源代码托管网站的Pages服务的区别

|名称| Github Pages |Gitee Pages|Coding Pages|
|—-|—-|—-|—-|—-|
|优点| 自由度高,可以自己绑定域名,可以自动更新 |访问速度快,有中文|没用过|
|缺点| 访问速度较慢 |自动更新和绑定域名需要购买200一年的高级版|没用过|
接下来就用GitHub Pages作示范

正式安装

环境准备

  • 系统 这里是Windows10 20H2 所以接下来的内容都是基于Windows的,如果你是Mac或者Linux用户,请参照官方文档进行安装
  • Node.js 版本12.0以上 我使用的是v14.7.0
  • Git 版本不限 我使用的是2.28.0
  • 终端 没什么太大要求,cmd,powershell都可以,如果遇到权限相关的问题 尝试用管理员身份打开再次执行

    安装cnpm

    注:这一步不是必须的,但是建议做一下,可以极大提高安装速度,如果你没有安装,这一段可以跳过,在接下来的命令中把cnpm替换成npm即可.

输入 npm install -g cnpm --registry=https://registry.npm.taobao.org
然后输入cnpm -v查看安装情况
如果你用的也是powershell,那这一步可能会出现如下问题

cnpm : 无法加载文件 C:\Users\29703\AppData\Roaming\npm\cnpm.ps1，因为在此系统上禁止运行脚本。有关详细信息，请参阅 https
:/go.microsoft.com/fwlink/?LinkID=135170 中的 about_Execution_Policies。
所在位置 行:1 字符: 1
+ cnpm
+ ~~~~
    + CategoryInfo          : SecurityError: (:) []，PSSecurityException
    + FullyQualifiedErrorId : UnauthorizedAccess

这时候,有两种解决办法

  1. 用cmd (推荐)
  2. 使用管理员身份打开终端
    输入 Set-ExecutionPolicy UnRestricted

然后再输入cnpm -v
不出意外,这样就可以运行了

正式部署hexo

输入cnpm install -g hexo-cli全局安装Hexo

然后输入

//这里的xxx是文件夹的名字 
hexo init xxx
cd xxx
cnpm install
hexo server

打开浏览器浏览localhost:4000
中途没报任何错误而且网页正常显示就算完成了.

基本操作

见官方文档 全部写起来浪费时间

装载主题

进入这个网站寻找你喜欢的主题 我这里用的主题是Fluid
进入网址 然后clone下来 你就能在当前文件夹见到他了
然后拖到博客文件夹/themes这个文件夹中
回到博客文件夹,修改_config.yml这个文件
找到Extensions 然后将下面的theme修改成你当前主题文件夹的名字,保存 再运行hexo s测试效果

其他修改

这些一般都在_config.yml和主题的_config.yml中

  • 网页标题 title:
  • 网页副标题 subtitle:
  • 所属者 author:
  • 语言 language: //中文是zh-CN
  • 时区 timezone:

如何修改主题的_config.yml一般都写在主题的readme.md里面

部署到GitHub

首先,新建一个仓库,仓库名称为你的用户名.github.io
然后设置你git的账号密码

git config --global user.name  "username"  
git config --global user.email  "email"
git config --global user.email "you@example.com"

接着安装插件cnpm install hexo-deployer-git --save
然后修改配置

deploy:
  type: git
  repo: <repository url> //你仓库的同步地址
  branch: [branch] //gh-pages

运行hexo clean && hexo deploy‘
输入一下你仓库的名字 就可以正常访问了
以后更新 就只要再执行一下上面的命令了
注:gitee的用户需要手动更新
